ATP Data

The data was obtained from Kaggle. This dataframe, as explained in the Data Card, contains information about every tennis match from 2013 to 2023 in male singles. It was downloaded on 07-05-2023.

Our objective is to choose a player and analyze his games to see if there are factors that can lead us to determine a certain probability that he will win the match. Since we are talking about guessing the outcome of a match, we have to keep in mind that it will be very difficult to obtain a perfect model, but at least we can hope to gain some insights as to what conditions may help the given player.

Looking for relevant data

We will use some plots to determine whether there might be a relation between the variables and de outcome of the match.

Series

The Series variable represents the category of tournament which is being played at. The main difference between the different categories is the number of points that each tournament awards to the winner. The most important are the Grand Slams, which give 2000 points to the winner; then the Masters1000, ATP500, and ATP250, which award 1000, 500 and 250 points, respectively. There is also the Masters Cup (which is now called ATP Finals), that only happens once a year and awards the points in a different way.

We can see that this variable can add some (admittedly low) information, which makes a bit of sense: he performs best at ATP250 tournaments, where the competition might not be as fierce, but he also has good results in Grand Slams. This can maybe be attributed to the fact that it is the most important tournament a tennis player can ever achieve to win, so it makes sense that he gives a little extra to win.

plot(table(train_atp$player_win,train_atp$Series), las = 2)

Court

This variable considers whether the court is indoors or outdoors. Apparently, Isner plays better when the court is outdoors, but the difference is not as noticeable.

plot(table(train_atp$player_win,train_atp$Court))

Surface

Tennis matches are played on three different surfaces: Hard, Grass, and Clay. The differences between each surface cannot be ignored. It is no secret that many players have clear prefference of one surface over another. The most famous example may be Nadal on clay, or maybe Federer on Grass. Isner is not the exception: he seems to perform better on clay courts, while he may not be as good on hard courts.

plot(table(train_atp$player_win,train_atp$Surface))

Round

The round is also important. It is generally easier to win in the first few rounds than it is to win later in the tournament.

plot(table(train_atp$player_win,train_atp$Round), las=2)

Best of

This variable represents how many sets the player has to win in order to win the match. Most tournaments are played to the best of 3 sets, except for the Grand Slams, which are played to the best of 5 sets. While there might be some difference in the performance of the player when it comes to the best of 3 or 5 sets, I believe that this difference is encompassed better by the Series variable, as the information of whether it is to the best of 3 or 5 is alredy contained in said variable. Besides, Best.of is a categorical variable encoded as a discrete variable, which is generally not good practice. By not taking this variable into account, we also avoid having to solve this problem.

Logistic regression

As we made in the Case Study IV, we assume a logistic regression for this problem. We set \(y_i = 1\) if Isner won the \(i\)-th match and \(0\) otherwise. Therefore,

\[y_i \mid p_i \sim \textrm{Bernoulli}(p_i),\] where \(p_i\) is the probability that Isner will win the \(i\)-th match such that \[\log{\frac{p_i}{1+p_i}} = \beta_0 + \beta_1 x_1^i + \cdots + \beta_r x_r^i\] where \(x_j^i\) is the value the \(j\)-th predictor takes for the \(i\)-th match and \(r\) is the number of predictors.

We assume proper but non informative priors for the parameters: \[ \beta_j \sim \textrm{N}(0,1000) \] We now use the MCMCpack to obtain a sample from the posterior of \((\beta_0, \ldots, \beta_r)\). As a reminder, we will use Surface, Round, log(Pts_Rival), log(Odd_Rival), and log(Odd_Player) as variables to analyze the outcome of a match. Even though Round and Series can be of importance, we don’t have enough data to estimate the parameters of the model.

library(MCMCpack)
out <- MCMClogit(player_win ~ Surface + log(Pts_Rival) + 
                   log(Odd_Rival) + log(Odd_Player), data=train_atp, 
                 burnin=1000, mcmc=21000, thin=10)

We could obtain the trace plots for each \(\beta_j\) using plot(out), but the plot won’t fit here. We can, however, obtain the posterior means, standard deviations and credible intervals with

Every match in our test dataset

new_SurfaceGrass <- ifelse(test_atp$Surface == "Grass", 1, 0)
new_SurfaceHard <- ifelse(test_atp$Surface == "Hard", 1, 0)
new_Pts_Rival <- log(test_atp$Pts_Rival)
new_Odd_Rival <- log(test_atp$Odd_Rival)
new_Odd_Player <- log(test_atp$Odd_Player)

correct_losses <- 0
total_losses <- 0
correct_wins <- 0
total_wins <- 0

for (i in 1:nrow(test_atp)) {
  win.prob <- inv.logit(out[,1] + new_SurfaceGrass[i]*out[,2] 
                      + new_SurfaceHard[i]*out[,3] + new_Pts_Rival[i]*out[,4]
                      + new_Odd_Rival[i]*out[,5] + new_Odd_Player[i]*out[,6])
  
  # If the model thinks it is a loss, we count
  if (quantile(win.prob, 0.975) < 0.5) {
    if (test_atp[i,]$player_win == 0) {
      correct_losses <- correct_losses + 1
    }
    total_losses <- total_losses + 1
  }
  
  # If the model thinks it is a win, we count
  if (quantile(win.prob, 0.025) > 0.5) {
    if (test_atp[i,]$player_win == 1) {
      correct_wins <- correct_wins + 1
    }
    total_wins <- total_wins + 1
  }
}

print(paste("Correct losses ratio:", correct_losses / total_losses))
## [1] "Correct losses ratio: 0.647058823529412"
print(paste("Correct wins ratio:", correct_wins / total_wins))
## [1] "Correct wins ratio: 0.773809523809524"
print(paste("Total hit ratio:", (correct_wins + correct_losses) /
              (total_wins + total_losses)))
## [1] "Total hit ratio: 0.752475247524752"

For a sports guess, the ratios don’t seem to be that bad. At least it is a better guess than to simply bet in favor of Isner all the time. The question now is how many times is the model so “sure” of the outcome of the match?

(total_losses + total_wins) / nrow(test_atp)
## [1] 0.6824324

This ratio is a bit small. The model only takes a guess less than 70% of the time. The good thing is we can lower our standards and force it to take a guess all of the time, or as much as we want. We could repeat the previous analysis using the posterior mean instead of quantiles:

correct_losses <- 0
total_losses <- 0
correct_wins <- 0
total_wins <- 0

for (i in 1:nrow(test_atp)) {
  win.prob <- inv.logit(out[,1] + new_SurfaceGrass[i]*out[,2] 
                      + new_SurfaceHard[i]*out[,3] + new_Pts_Rival[i]*out[,4]
                      + new_Odd_Rival[i]*out[,5] + new_Odd_Player[i]*out[,6])
  
  # If the model thinks it is a loss, we count
  if (mean(win.prob) < 0.5) {
    if (test_atp[i,]$player_win == 0) {
      correct_losses <- correct_losses + 1
    }
    total_losses <- total_losses + 1
  }
  
  # If the model thinks it is a win, we count
  if (mean(win.prob) >= 0.5) {
    if (test_atp[i,]$player_win == 1) {
      correct_wins <- correct_wins + 1
    }
    total_wins <- total_wins + 1
  }
}

print(paste("Correct losses ratio:", correct_losses / total_losses))
## [1] "Correct losses ratio: 0.6"
print(paste("Correct wins ratio:", correct_wins / total_wins))
## [1] "Correct wins ratio: 0.731481481481482"
print(paste("Total hit ratio:", (correct_wins + correct_losses) /
              (total_wins + total_losses)))
## [1] "Total hit ratio: 0.695945945945946"
(total_losses + total_wins) / nrow(test_atp)
## [1] 1

It is no surprise that every ratio went down, but maybe not as much. It still is a better guess than to simply bet in favor of Isner all the time.
